<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → published bundle | Workflow execution trust; malicious action versions could exfiltrate or inject |
| Pages CDN → end user | Stale cache could serve outdated bundle |
| Browser at deployed URL → IndexedDB | Same-origin policy is the only isolation between this app and other apps on the github.io subdomain |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-13 | Tampering | GitHub Actions third-party action versions | medium | mitigate | Pin all actions to MAJOR version tag (@v4, @v3) — these resolve to the latest patched minor; Dependabot would auto-bump if enabled. All actions listed are official GitHub-owned (actions/* namespace) — no third-party actions in the workflow. |
| T-01-14 | Spoofing | Pages source set to wrong branch | low | mitigate | Pages source explicitly set to "GitHub Actions" in repo settings (user_setup dashboard_config item); workflow uses `actions/deploy-pages` artifact path NOT a `gh-pages` branch (RESEARCH.md). Misconfiguration surfaces as deploy step failure, not silent wrong-bundle serve. |
| T-01-15 | Information disclosure | Source map reveals comment-style secrets | high | mitigate | DEP-03 requires no credentials in source; plan 01 acceptance criterion greps for `localStorage`/`fetch(`/`.env*` etc.; this plan adds manual network-tab inspection on the deployed URL (Task 3 step 10) to confirm no outbound requests to credential-bearing endpoints. No `.env*` files committed (plan 01 .gitignore). |
| T-01-16 | Denial of service | Multi-tab IndexedDB upgrade blocked | medium | mitigate | `db.on('versionchange', () => db.close())` registered in plan 02 src/storage/db.ts per RESEARCH.md Pitfall 3. Plan 03 manual phone test only exercises one tab so does not stress this path — Phase 2 dashboard work will validate multi-tab when schema version advances. |
| T-01-17 | Tampering | Cached stale index.html on GitHub Pages CDN | low | accept | Vite content-hashes JS/CSS; index.html is not content-hashed but GitHub Pages CDN TTL is short. Hard refresh resolves on the rare miss. Adding `_headers` Cache-Control: no-cache for index.html is a v2 improvement if observed in practice. |
| T-01-18 | Repudiation | No deploy log retention | low | accept | GitHub Actions retains logs 90 days by default; sufficient for a single-user personal app. No regulatory retention requirement. |
</threat_model>
