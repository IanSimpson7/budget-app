## Surface & Background Decision

**Primary background: `#161210`**

Justification: R=22 G=18 B=16 — red channel leads, green and blue trail below neutral gray parity. This produces a warm charcoal with brown-amber undertone that reads as wood ash, not concrete. `#0f0f0f` is a cool near-black that neutralizes the warmth; `#1a1410` overshoots into reddish-brown. `#161210` is the correct register — dark enough for strong contrast, warm enough to carry the "made by a person" tone.

**Accent: `#c0522a` (rust/burnt orange)**

Justification: Sits between brief's `#b54a1e` lower bound and warm amber `#c47a1e`. Rust reads as decisive and craft-made; amber risks convention-confusion with warning states (yellow-orange is universal caution); cream off-white lacks the "bite" the brief calls for. `#c0522a` against `#161210` achieves a contrast ratio of approximately 5.4:1 — passes WCAG AA for large text and interactive elements. Reserved strictly for the elements listed in the Color section below.

## Spacing Scale

All values multiples of 4. Tailwind custom token names prefixed `sp-` to distinguish from Tailwind defaults.

| Token | px Value | Tailwind class suffix | Usage |
|-------|----------|-----------------------|-------|
| xs | 4px | `sp-1` | Icon gaps, inline label padding, tight stacks |
| sm | 8px | `sp-2` | Compact element spacing, input inner padding (vertical) |
| md | 16px | `sp-4` | Default element spacing, section inner padding |
| lg | 24px | `sp-6` | Section padding, card interior |
| xl | 32px | `sp-8` | Layout column gaps |
| 2xl | 48px | `sp-12` | Major section breaks |
| 3xl | 64px | `sp-16` | Page-level vertical spacing (desktop only) |

Exceptions:
- Touch targets (buttons, inputs, nav items): minimum height 44px, minimum tap area 44×44px — enforced by `min-h-[44px]` and `py-[10px]` (10px top + 10px bottom + 24px content = 44px) on all interactive elements. This is a mobile-first hard requirement from CONTEXT.md §Specifics.
- Input horizontal padding: 12px (`sp-3`) — not on the 8-point scale, but justified for number input legibility at mono font size. This justification must not be removed.

## Typography

Three typefaces only. All available on Google Fonts.

| Typeface | Role | Where |
|----------|------|--------|
| DM Serif Display | Headings, prominent labels, section headers | App name, page titles, surface headers |
| IBM Plex Mono | All financial data | Every number, amount, date, currency value, input field |
| IBM Plex Sans | Body text, supporting labels, helper text, toasts | Everything not heading or financial |

### Type Scale

| Role | Font | Size | Weight | Line Height | Tailwind classes |
|------|------|------|--------|-------------|-----------------|
| Display (app name / phase header) | DM Serif Display | 28px | 400 (Regular — DM Serif is display-weight by design) | 1.15 | `font-display text-[28px] leading-[1.15]` |
| Heading (surface title, e.g. "Settings") | DM Serif Display | 20px | 400 | 1.2 | `font-display text-[20px] leading-[1.2]` |
| Label (field labels, nav labels) | IBM Plex Sans | 14px | 600 (SemiBold) | 1.4 | `font-sans text-sm font-semibold leading-[1.4]` |
| Body / helper | IBM Plex Sans | 14px | 400 (Regular) | 1.5 | `font-sans text-sm leading-[1.5]` |
| Financial value (amounts, dates) | IBM Plex Mono | 16px | 400 (Regular) | 1.4 | `font-mono text-base leading-[1.4]` |
| Financial value — prominent (input display, key figures) | IBM Plex Sans | 20px | 600 (SemiBold) | 1.3 | `font-sans text-[20px] font-semibold leading-[1.3]` |

Note on prominent financial value: IBM Plex Mono does not have a 600-weight variant. To maintain the two-weight system (400 and 600 only), the prominent financial value row uses IBM Plex Sans at 600 rather than introducing weight 500. The semantic intent — "this number is important" — is carried by size (20px vs 16px) and weight (600 vs 400), which is sufficient visual differentiation. Mono rendering is retained for all standard financial values (16px, weight 400); the prominent size shifts to IBM Plex Sans to honour the weight constraint.

Tabular figures: IBM Plex Mono uses tabular figures by default — no `font-variant-numeric: tabular-nums` override needed. Confirm with `font-feature-settings: "tnum"` in Tailwind config as a belt-and-suspenders.

Max declared sizes: 28px, 20px, 16px, 14px — four sizes, two font weights: 400 (Regular) and 600 (SemiBold) only.

## Color

All hex values are exact. No opacity variants on primary surfaces (opacity produces glassmorphism; use solid colors from the palette instead).

| Role | Hex | Tailwind token | Usage |
|------|-----|----------------|-------|
| **Surface / dominant (60%)** | `#161210` | `bg-surface` | Main page background |
| **Surface raised (30%)** | `#201c19` | `bg-surface-raised` | Cards, Settings panel, Backup panel, input backgrounds |
| **Surface border** | `#332d28` | `border-surface-border` | All component borders — hard edges, no shadow substitute |
| **Text primary** | `#e8dcc8` | `text-primary` | All primary text — warm off-white, not pure white |
| **Text secondary** | `#9a8e82` | `text-secondary` | Helper text, supporting labels, placeholders |
| **Text disabled** | `#5a524c` | `text-disabled` | Disabled states |
| **Accent** | `#c0522a` | `bg-accent` / `text-accent` / `border-accent` | (see reserved list below) |
| **Accent hover** | `#d4633a` | `bg-accent-hover` | Accent button hover state only |
| **Destructive** | `#a63228` | `bg-destructive` / `text-destructive` | Destructive actions only (import replace-all confirmation) |
| **Destructive hover** | `#bf3e34` | `bg-destructive-hover` | Destructive button hover |
| **Success** | `#4a7c59` | `text-success` | Import success confirmation only |
| **Warning** | `#9a7a2e` | `text-warning` | Reserved for Phase 4 unpriced-ingredient flag; declare now |

**Accent `#c0522a` reserved for:**
1. Primary CTA button (Export backup, Save settings) — background fill
2. Active nav indicator (underline or left border on active route)
3. Focus ring on interactive elements (2px solid, 2px offset)
4. No other elements. Not for icons, not for decorative elements, not for borders in non-interactive contexts.

**No gradients. No shadows on cards or surfaces.** All depth signaled via border `#332d28` (1px solid) and background step (`#161210` → `#201c19`).

---

## Border Radius

Hard edges as default. Creative brief: "hard edges or max 2–4px border-radius."

| Element | Radius | Tailwind |
|---------|--------|---------|
| Cards, panels | 2px | `rounded-sm` |
| Buttons | 2px | `rounded-sm` |
| Inputs | 2px | `rounded-sm` |
| Toast notifications | 2px | `rounded-sm` |
| Focus rings | 2px offset, 2px radius | `outline-2 outline-offset-2` |
| No element exceeds 4px | — | Never `rounded-md` or larger |

No pill buttons. No rounded-full interactive elements.

---

## Shadow Treatment

No box shadows on cards or layout surfaces. Component definition comes from borders only.

Exception allowed: focus ring shadow (`box-shadow: 0 0 0 2px #c0522a`) as accessibility affordance — this is a functional ring, not decorative depth.

## Interaction States

All interactive elements must have explicit styles for: default, hover, focus-visible, active, disabled.

No visual difference between touch and mouse (single implementation covers both — use `:hover` and `:focus-visible`).

Input number validation: passive floor and defended line must be > 0; food floor seed must be > 0. Show inline error below input (IBM Plex Sans, 14px, `text-destructive`) — never block the save action silently.

Import flow state machine:
1. Default: Secondary "Import backup" button
2. File picker open (browser native): no custom UI
3. File selected: show replace warning + "Replace and import" (destructive) + "Cancel import" (secondary)
4. Processing: button shows "Importing..." disabled state, spinner (lucide Loader2 icon, 16px, animate-spin)
5. Success: toast success, state reset to default
6. Error: toast error with specific message, state reset to default

## Phase Scalability Notes

These decisions must hold through Phase 5. Executor must not override them in later phases without a design revision.

- The Settings surface will gain additional fields in Phases 2–3 (income floors, expense parameters). The panel pattern (card, full-width inputs, single Save settings CTA) scales by adding fields — do not introduce tabs or nested navigation within settings.
- Phase 2 introduces the dashboard. Dashboard uses the same surface/raised color system. Financial data on the dashboard MUST use IBM Plex Mono — no exceptions for "cleaner" sans presentation of numbers.
- Phase 4's locked food floor UI (FOOD-12: no downward-edit affordance) should use `text-disabled` color + a locked icon (lucide `Lock`, 14px) to signal read-only without using a disabled input that implies system-level unavailability.
- The accent `#c0522a` is reserved. Phase 2–5 features should NOT introduce new accent colors. Semantic colors (success, warning, destructive) are already declared.
- The destructive color `#a63228` is used only for the import replace-all confirmation in Phase 1. In Phase 2+, it applies to: deleting income entries, deleting expense line items, clearing EF sweep records. Pattern: always require explicit confirmation before a destructive action executes.
