## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pasted free text → parser → confirm → storage | Untrusted text is parsed, reviewed by the user, then only checked rows persist |
| (no network / no credentials) | Local-only, C2/C3 — N/A categories below |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-D | Denial of Service | PasteParseFlow / parser | mitigate | Parser (02-02) caps input length + anchored regexes; parse is synchronous and linear. Empty result returns the inline error path, never a hang/crash. |
| T-02-03 | Integrity | conservative auto-check | mitigate | D-05: only PAYROLL/known-source credits default-checked; gold-fixture test proves the $3,000 asset sale + gift are excluded from the committed set, protecting floor math (and the spirit of SURP-06). |
| T-02-X | (XSS) | source/note render in ConfirmTable + toast | mitigate | React auto-escapes the parsed `source`/`note`; never `dangerouslySetInnerHTML`; toast copy is a fixed template ("Saved {N} checks.") and never concatenates parsed content (BackupPage rule). |
| T-02-06 | Tampering (spec) | commit path | mitigate | Commit uses `storage.addIncomeChecks`/`saveKnownSources` only; no money-move/credential method (C2/C3). |
| T-02-S/R/I | Spoofing/Repudiation/Info-Disclosure | — | N/A | Single-user local app, no auth/sessions/secrets. |
</threat_model>
