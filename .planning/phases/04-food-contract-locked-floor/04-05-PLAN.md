## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config UI input to storage | Unit-cost/portion/flavor edits cross from the UI into persisted state (validated in Plan 01) |
| rendered floor to user | The displayed floor is the C1-sensitive surface; any edit affordance on it would be a breach |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Tampering | floor display element | mitigate | The floor value renders in a non-interactive span with a Lock icon; no input/pencil/edit affordance exists on the floor line (V6 page test asserts no input in the floor card). C1. |
| T-04-15 | Information Disclosure | gap detail copy | accept | Gap copy names ingredients/meals (no PII, no financial secrets); local-only render. No mitigation needed beyond the existing local-only model (C2). |
| T-04-16 | Repudiation | silent gap | mitigate | Every uncertainty (unpriced ingredient, undefined meal, staleness) is surfaced via the amber badge and an individually-named detail row (D-11); color is never the sole differentiator (a11y) so a gap can never be invisible. C1. |
| T-04-17 | Tampering | downward-framing copy | mitigate | The C1 NEVER-USE strings are grep-asserted absent from FoodPage.tsx; the config heading is fixed to accuracy framing — no UI path suggests cutting the floor. C1. |
</threat_model>
