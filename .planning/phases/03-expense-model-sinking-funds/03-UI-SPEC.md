## Surfaces

### Surface 1: Dashboard Addition — Survival Floor Metric Card

**Route:** `/dashboard` (existing page, new card appended to the metric grid)

**Placement:** Fourth card in the existing `grid grid-cols-3` row, OR the grid becomes 2-row (2 top + 2 bottom) if a 4-card layout is preferred. Executor decision — either layout is acceptable; the 3-col grid already exists. The survival floor card MUST NOT replace an existing card; it is additive.

**Component:** `MetricCard` (existing, zero modification required)

Props:
```
label="Survival floor"
value={currency.format(survivalFloor)}   // font-mono 20px — from survivalFloorAtom
subtext="fixed + food seed"              // optional orientation label
variant="default"
```

**Atom:** `survivalFloorAtom` — derived Jotai atom, recomputes when `expenseItems` or `floors.foodSeed` change. Formula: `Σ(PROTECTED expense lines, monthly-normalized) + Σ(sinking-fund monthlyAccruals) + floors.foodSeed`. Never stored; always derived (FOUND-06).

**Empty state:** When no expense lines exist yet, value renders as `$0.00` with `valueColor="text-text-disabled"`. No separate empty-state message — the zero value is self-explanatory; full empty state lives on `/expenses`.

### Surface 2: `/expenses` Route

**Route:** `/expenses` — new HashRouter entry alongside `/dashboard`, `/entry`, `/settings`, `/backup`.

**Nav link:** "Expenses" added to `AppShell` nav (same `navClasses` pattern, same `min-h-[44px]`).

**Page layout:** `flex flex-col gap-sp-6 max-w-[640px] mx-auto` (slightly wider than entry's 480px to accommodate the two-column categorized view).

**Page heading:** `<h2 className="font-display text-[20px] leading-[1.2] text-text-primary">Expenses</h2>`

#### Sub-section A: Add Expense Form

Inline form at the top of the page (no modal). Fields:

| Field | Input type | Constraint |
|-------|-----------|------------|
| Name | Text input | Required, `font-sans text-sm`, border `surface-border`, focus `border-accent` |
| Amount | NumberInput (existing component) | Required, dollar value |
| Cadence | Select (styled to match input) | Values: Monthly / Annual / One-off |
| Classification | Segmented toggle or two radio buttons | PROTECTED / GATEABLE — mutually exclusive (D-02) |

Classification toggle spec:
- Two buttons side by side, `min-h-[44px]`.
- Active state: `bg-surface-raised border border-accent text-text-primary font-semibold`.
- Inactive state: `bg-transparent border border-surface-border text-text-secondary`.
- No third state. Default: PROTECTED selected.
- ARIA: `role="group"` with `aria-label="Classification"`, each option `role="radio"` or `aria-pressed`.

CTA: PrimaryButton "Add expense" — disabled until name + amount both filled.

Annual/one-off cadence: helper text under the amount field: "Counted as [amount ÷ 12]/mo in the survival floor." (only shown when cadence is Annual or One-off for one-off lines, one-off is excluded from floor — adjust copy: for one-off, helper = "One-off costs are excluded from the survival floor.")

#### Sub-section B: Categorized Expense List (SC#1)

Two-column layout on sm+ (`grid grid-cols-2 gap-sp-4`), single column on mobile.

**Column 1 — PROTECTED**
- Column heading: `font-sans text-xs font-semibold text-text-secondary uppercase tracking-wide` — "Protected"
- Each line item: `bg-surface-raised border border-surface-border rounded-sm p-sp-3`
  - Name: `font-sans text-sm text-text-primary`
  - Amount: `font-mono text-[20px] text-text-primary` + cadence badge (see below)
  - Edit button: `SecondaryButton` or inline icon-text "Edit" at `text-xs`, `min-h-[44px]` tap area
  - Delete button: `DestructiveButton` — triggers inline confirmation (see Copywriting)

**Column 2 — GATEABLE**
- Same structure as PROTECTED column, different heading label.

**Cadence badge:** Small inline label to the right of the amount. `font-sans text-xs text-text-secondary`. Values: "/ mo", "/ yr", "one-off". Renders inline — e.g. `$1,300 / mo`.

**Row edit (inline, not modal):** Clicking "Edit" on a row replaces the row with an inline mini-form (same fields, pre-populated). "Save" and "Cancel" buttons. No modal overlay — consistent with the typed-only, no-friction posture of Phase 3.

**Seeded starter rows (D-03):** Housing all-in $1,300/mo · Electric ~$65/mo · Fuel ~$238/mo · Claude ~$100/mo — all PROTECTED. These appear on first run as editable rows. Executor to implement as "seed if PROTECTED expense rows are empty" idempotent check (not a settings flag).

**EXP-07 guard (visual):** No UI affordance for adding Whey/supplement under expenses. If a user types "whey" or "supplement" in the name field, show helper text: "Supplement costs belong in the food floor (Phase 4). Add here only if you want it in fixed-ex-food — this will feed the survival floor." This is a soft advisory, not a hard block.

### Surface 3: `/funds` Route

**Route:** `/funds` — new HashRouter entry.

**Nav link:** "Funds" added to `AppShell` nav.

**Page layout:** `flex flex-col gap-sp-6 max-w-[640px] mx-auto`

**Page heading:** `<h2 className="font-display text-[20px] leading-[1.2] text-text-primary">Sinking Funds</h2>`

#### Fund Card

Each sinking fund renders as a card. Layout: `bg-surface-raised border border-surface-border rounded-sm p-sp-4 flex flex-col gap-sp-3`

Card contents (top to bottom):

1. **Fund name:** `font-sans text-sm font-semibold text-text-primary`
2. **Status badge:** `font-sans text-xs font-semibold` — "On track" in `text-success` OR "Behind" in `text-warning`. Inline next to name on the same row (`flex justify-between items-center`).
3. **Progress bar:** `w-full h-[6px] rounded-sm bg-surface-border` container; inner fill `bg-success` (on-track) or `bg-warning` (behind). Width = `(balance / annualAmount) × 100%`, capped at 100%. `role="progressbar" aria-valuenow={balance} aria-valuemin={0} aria-valuemax={annualAmount}`.
4. **Metrics row:** Three values side by side (`grid grid-cols-3 gap-sp-2`):
   - Balance: label "Current" + value `font-mono text-[20px] text-text-primary`
   - Target: label "Target" + value `font-mono text-[20px] text-text-primary`
   - Monthly: label "/ mo accrual" + value `font-mono text-[20px] text-text-primary`
5. **Payout date:** `font-sans text-xs text-text-secondary` — "Due [Month YYYY]"
6. **Provisional flag (car insurance):** If `annualAmount` is flagged provisional, show `font-sans text-xs text-warning` — "Target is provisional — update at renewal."
7. **Actions row:** "Edit" (`SecondaryButton text-sm`) · "Mark paid" (`SecondaryButton text-sm`) · "Delete" (`DestructiveButton text-sm`). All `min-h-[44px]`.

**Mark paid interaction (EDGE-06 / D-07):**
- For `cadence: 'annual'` funds: inline confirmation "Mark car insurance as paid? Balance resets and payout date advances to [YYYY+1]-MM." Confirm with PrimaryButton "Mark paid", cancel with SecondaryButton "Cancel".
- For `cadence: 'oneoff'` funds: inline confirmation "Mark [name] as complete? This fund will be archived." Confirm with PrimaryButton "Mark complete".
- No modal — inline replace-with-confirmation pattern (same as expense delete below).

#### Add Sinking Fund Form

Inline form below the fund cards (or top of page if no funds exist). Fields:

| Field | Input type | Constraint |
|-------|-----------|------------|
| Fund name | Text input | Required |
| Annual target | NumberInput | Required, dollar value |
| Monthly accrual | NumberInput | Pre-populated as `annualTarget / 12`, editable |
| Current balance | NumberInput | Default 0, required |
| Payout date | Month input (`type="month"`) | Required, YYYY-MM |
| Cadence | Select | Annual / One-off |

CTA: PrimaryButton "Add fund" — disabled until name + target + payout date filled.

Helper under "Annual target": "Provisional targets can be updated anytime." (shown always for the car-insurance use case; applies generically).

#### Empty State (no funds)

Shown when `sinkingFunds` list is empty:

```
Heading: "No sinking funds yet"
Body:    "Add a fund to spread annual costs across months — no budget shock when the bill arrives."
CTA:     "Add your first fund" — scrolls to or focuses the Add form
```

## Interaction States

All interactive elements must implement all four states:

| State | Implementation |
|-------|---------------|
| Default | As specified per component |

| Disabled | `disabled:opacity-40 disabled:cursor-not-allowed` |
| Active | `active:scale-[0.98]` on PrimaryButton only |

## Accessibility

- All form inputs have associated `<label>` elements (htmlFor + id pair).
- Progress bars: `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`.
- Classification toggle: `role="group"` + `aria-label="Classification"` on wrapper; individual options use `aria-pressed` or `role="radio"`.
- Inline confirmation (delete/mark-paid): manages focus — on confirmation open, first focusable button inside confirmation receives focus. On cancel, focus returns to the triggering button.
- Status badges ("On track" / "Behind") use `text-success` / `text-warning` — do NOT rely on color alone; the text label carries the semantic meaning.
- Nav: new links inherit the existing `navClasses` focus-visible pattern with `outline-accent`.
