## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSON import file → storage.replaceAll | Untrusted file content crosses into IndexedDB; only network/credential-free surface in the app (C2) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | importAll/replaceAll on crafted backup (annualAmount/amount = NaN, negative, or non-finite) | mitigate | Validate every financial field with `Number.isFinite` in replaceAll's expense/fund insert loops; reject (throw ImportError-style) on non-finite. Round-trip test covers happy path; a malformed-value test covers rejection. |
| T-03-02 | Tampering | addExpenseItem / addSinkingFund writing a NaN amount that propagates to survivalFloorAtom | mitigate | Guard at storage write: `if (!Number.isFinite(amount)) throw`. Unit test asserts the throw. |
| T-03-03 | Tampering | Hypothetical money-movement / credential method added to storage surface (C2/C3 erosion) | mitigate | Structural: add NO such method. The existing storage.test.ts absence-proof (saveCredentials/moveMoney/executeSweep/decreaseFoodFloor undefined) must keep passing — re-run it in this plan's verify. |
| T-03-04 | Tampering | Prototype pollution via `__proto__` keys in imported settings object | accept | Settings are written via `db.settings.put({key,value})` with explicit keys; Dexie does not eval keys. Low risk for a local single-user app with no network surface (C2). Documented, not mitigated further. |
</threat_model>
