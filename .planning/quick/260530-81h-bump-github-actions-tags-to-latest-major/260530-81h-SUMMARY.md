# Summary: Bump GitHub Actions tags to latest majors (Node 24)

## What changed

`.github/workflows/deploy.yml` — five action tags bumped to current latest
majors (verified live via `gh api repos/<action>/releases/latest` on 2026-05-30):

| Action | Old | New |
|--------|-----|-----|
| actions/checkout | v4 | v6 |
| actions/setup-node | v4 | v6 |
| actions/configure-pages | v4 | v6 |
| actions/upload-pages-artifact | v3 | v5 |
| actions/deploy-pages | v4 | v5 |

## Why

GitHub force-migrates Actions runners to Node 24 on 2026-06-02; the old
`@v4`/`@v3` majors ran on Node 20 and would fail after migration. Closes the
STATE.md dated follow-up [ACT BEFORE 2026-06-02].

## Verification

- `git diff` confirmed exactly five tag changes; no structural YAML changes.
- Workflow logic (test-gate before build, artifact deploy flow, Pages source)
  unchanged.
- Build-time Node version (`.nvmrc`) untouched — independent of runner Node.
- End-to-end green deploy will be confirmed on the next push to `main` (this
  docs commit triggers it).

## Surfaced and fixed (deploy gate)

This was the **first deploy since Phase 1** — all of Phase 2/3/4-planning was
committed locally but never pushed, so CI had not run the test gate over Phase
2+ code. The push exposed one pre-existing failure:

- `IncomeBar.test.tsx > renders defended line label "$3k"` failed in CI only.
- **Root cause:** [IncomeBar.tsx](../../../src/components/IncomeBar.tsx) built the
  defended-line label via `Intl` compact-currency, whose trailing-zero behavior is
  ICU-version-dependent. CI ran `.nvmrc` Node 20 → `"$3.0k"`; local/Node 24 →
  `"$3k"`. The assertion expected `"$3k"`.
- **Fix (commit `0010216`):** strip a trailing `.0` before the compact unit so the
  label is ICU-stable; **bump `.nvmrc` 20→24** (Node 20 hit EOL 2026-04-30; local
  and the new runners are on 24; all 212 tests pass there).
- **Result:** deploy run `26681076762` green end to end — every bumped action plus
  build/deploy succeeded.
